# Develop Companion

Use this companion when `omc-develop` needs more execution discipline than the core skill body should carry.

## Execution quality bar

- Prefer the smallest coherent diff.
- Reuse repo-local patterns before adding abstractions.
- Keep unrelated cleanup out of scope.
- Name exactly what was verified.

## Common failure modes

- Hidden scope growth
- Verification deferred to review
- New abstractions added without a current need
