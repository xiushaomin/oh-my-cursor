# Plan Companion

Use this companion when `omc-plan` needs more sequencing or proof detail.

## Plan quality bar

- Milestones should be reviewable checkpoints.
- Dependencies should be explicit.
- Acceptance criteria should prove completion, not restate activity.
- Verification should point to likely repo-local checks when known.

## Common failure modes

- Buckets of activity with no proof shape
- Missing ownership or ordering
- Risks named vaguely instead of concretely
