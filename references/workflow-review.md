# Review Companion

Use this companion when `omc-review` needs a sharper findings-first pass.

## Review quality bar

- Findings come before summary.
- Every meaningful issue should point to evidence.
- Missing proof is itself a finding when risk is real.
- Handoff notes should stay minimal and evidence-backed.

## Common failure modes

- Turning review into implementation advice
- Vague findings with no file or behavior anchor
- Summary overshadowing the actual issues
