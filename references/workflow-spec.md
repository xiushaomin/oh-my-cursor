# Spec Companion

Use this companion when `omc-spec` needs more shape than the core skill body should hold.

## Spec quality bar

- Keep the scope boundary explicit.
- Make acceptance criteria observable.
- Keep the quality contract specific to the work.
- Remove trivia from open questions.

## Common failure modes

- Mixing multiple directions into one spec
- Describing implementation instead of requirements
- Writing generic quality promises that do not guide planning
